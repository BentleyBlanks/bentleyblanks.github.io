六边形邻接地貌
资产生产规范

面向 Forest、Hills、Mountain 三类 2.5D 地貌。最终交付 3 地形 × 4 季节 = 12 张透明图集；每张包含完整 64 个六邻接 mask，并与现有 dual-grid 地表共同工作。

本文目录适用边界文件与几何掩码版式美术语义提示词模板生产流程验收门槛

先把两种邻接问题分开

地表与抬升地貌不是同一个素材合同，不能再把“每片看三个格”误写成“所有素材只有三种情况”。

Ground：对偶三角网

flat-top odd-q 六边网格的 dual-grid 三角片，每片读取三个格心。每种地表实际需要 7 个非空掩码 × EAST/WEST = 14 槽，负责草地、土壤、基岩和未侦查边界的平面过渡。

Feature：完整六邻接

Forest、Hills、Mountain 必须知道六条公共边是否继续，因此使用 6 bit mask，共 64 槽。它负责林缘、丘脊和山脊；不能回退成每格一个圆底盆景。

不在本合同内：河流、普通道路、铁路需要自己的逐边连接合同；村庄、建筑和单位仍是实体。不能用 Feature 图集代替这些系统。

12 张正式图集

当前可运行基线：以下 12 个目标路径均已预烘焙并安装程序化 v1 PNG（3 类地貌 × 4 季节），运行时直接加载这些确定性资产，不再在启动时临时生成。它们用于验证 64-mask、socket、季节切换、缓存与性能合同；仍不是 Provider/Lovart 产出的正式美术图集。

单格

192 × 240 px，透明 RGBA。逻辑格心 (96,176)，锚点 (0.5, 176/240)，运行时固定绘制为 2.4 × 3.0 hexSize。不允许逐格缩放、旋转或随机镜像。

整图

8 × 8，原始 PNG 1536 × 1920 px。运行时会给每格加 8 px gutter 并生成 mip，活动纹理约为 1664 × 2048 px；只驻留当前季节三张。

六个 socket

bit方向单格像素opposite
0SE(156, 202)3 · NW
1S(96, 228)4 · N
2SW(36, 202)5 · NE
3NW(36, 150)0 · SE
4N(96, 124)1 · S
5NE(156, 150)2 · SW

bit=1 时，形体必须以固定宽度、色板和 alpha 到达该 socket；bit=0 时，该 socket 周围至少 10 px 保护区必须透明。相反方向在世界坐标中必须重合，误差不超过 1 px。

mask → 图集格位

direction bits = [SE, S, SW, NW, N, NE]
mask = Σ(connected[direction] ? 1 << direction : 0)
cell.x = mask & 7
cell.y = mask >> 3

下表按最终 8×8 版式生成。数字是 mask，文字是必须打开的接口；0 是孤立件，63 是六面腹地。

允许按 14 类拓扑语义组织生产，但最终必须烘成 64 个屏幕方向正确的格位。不要把完成的 2.5D raster 旋转 60°：树会倾倒，山体光向也会旋转。镜像同样会反转固定光源。

三种地貌的画面语义

Forest

透明底的竖直单树、灌木与林下暗部。边缘稀、腹地密；连接 socket 可由共用边缘树或很窄的林下带完成。单格视觉目标约 3–7 株，禁止整片花园状圆底。

Hills

低矮、不闭合的丘脊：东南侧宽而淡的阴影，西北侧窄高光，少量内部等高暗线。不得画实心草垛、完整椭圆圈或六边底板。

Mountain

连续山脚骨架 + 1 个主峰及必要肩峰。主轴跟随连接方向，边缘低、腹地高；固定左上光、右下阴影。禁止每格同一个完整圆底山包。

共同要求

三分之四俯视正交、低分辨率手绘像素质感、有限色板、硬边像素、统一暖色自然光；四周透明，不带文字、UI、旗帜、道路、河流、建筑或单位。

四季只改自然状态，不改拓扑与镜头

  • Spring：湿润新绿、少量花色，山石偏暖灰。
  • Summer：深绿与较强叶冠，土石略干。
  • Autumn：赭黄、棕红、橄榄绿混合，保持相同接口宽度。
  • Winter：低饱和灰绿、枯枝与克制积雪；积雪不能填满透明保护区。
绝对禁止：圆形/椭圆形地台、完整六角底座、浅色贴纸边、背景色块、白边或品红边；禁止在相邻 mask 间改变 socket 位置、宽度、色板或 alpha。

生产顺序与当前状态

  1. 在「BTL 单元格」选择 Features、地形和季节，导出当前 64-mask 程序化图集作为版式与 socket 参考。
  2. 按同一完整提示词生成两个候选：Lovart Unlimited 的 generate_image_gpt_image_2 必须处于 low 1K；同时用 generate_image_nano_banana_pro 以 1K 生成对照。保留两份输出并比较。
  3. 人工选择更符合固定镜头、透明边与地貌语义的候选；不得只因细节多就选中破坏接口的一张。
  4. 专用处理器切分/抠图/组装 64 格，补 gutter 前先执行 socket、透明区、尺寸和圆底检测。
  5. 在 Spring 与 Winter 的 1920×1080 固定种子地图中检查默认、0.75×、1.35× 三档缩放，再发布正式 PNG 与 QA 哈希。
截至 2026-07-19，12 张程序化 v1 PNG 已预烘焙安装；Provider/Lovart 正式 8×8 资产管线仍未接入。12 个 cell.hexfeature.* asset ID 目前仅为合同保留项，尚未注册为可生成任务；registry、专用 processor 与 ready validator 均未完成。本次程序化首版没有提交 Provider 请求、没有调用 Lovart，也没有消耗任何生成额度。在这条管线接通前，不要用旧 GeneratedAtlasProcessor 伪装处理，也不要发起 regenerate。

自动与人工验收

  • 文件名、1536×1920 尺寸、8×8/64 格和 RGBA 全部正确；单格角落透明。
  • 64 个 mask 与格位双射；相邻连接 bit 互为 opposite；bit=0 保护区透明。
  • 每一对相反 socket 的世界落点误差 ≤ 1 px，颜色/alpha 无可见跳变。
  • 完整圆形或椭圆形整格底座数量为 0；无白边、黑矩形、品红残边和 mip 串色。
  • Forest 默认视野每格约 3–7 株；Mountain 腹地以 1 个主峰为主，端头可有肩峰；Hills 不形成粗线网。
  • 村庄、铁路、单位、名字和选择框仍可读；未探明格不会通过 Feature 泄露地形。
  • 清缓存重建两次，atlas hash、画家条目和截图保持确定；同季运行时活动 Feature 图集不超过三张。
  • 1920×1080 桌面与约 390×844 移动宽度下，本页表格可横向滚动、正文不溢出。
