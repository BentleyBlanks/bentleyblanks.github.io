# 《角标清理铺》美术 / 音效生产管线

> 目标：消除当前同屏 5 种互斥画风造成的"违和感"，把所有视觉/听觉素材收敛到**唯一锚点风格 —— 温暖手绘 2.5D**（以 `assets/art/realistic-shop-counter.png` 为基准）。
>
> 本目录是给**多 Agent 协作 + 自我验收**用的"单一事实来源"。每个 Agent 在生成任何素材前，必须先读 `ART_BIBLE.md`（画风约束）与对应的 `prompts/*.md`（逐素材提示词），产出后必须按 `QA-PROTOCOL.md` 自检并循环，直到达标才算完成。

## 执行优先级（违和感从重到轻）

| 优先级 | 范围 | 原因 |
|---|---|---|
| **P0** | App 图标、弹窗事件图、UI 主题 | 当前"塑料糖果 3D + C4D 渲染 + 深色玻璃"三种语言冲突最强，是违和感主因 |
| **P1** | 升级图标、技能图标、顾客重绘 | 升级/技能图标尚不存在（直接按规范新建）；顾客是日系动漫贴图，需降光泽、对齐光照与调色板 |
| **P2** | 手机/变体、壁纸、品牌物料 | 当前可接受或影响面小，最后处理 |

## 给编排 Agent 的最短执行说明

1. **读规范**：`ART_BIBLE.md` + `prompts/00-global-style-block.md`。
2. **取任务**：从 `prompts/0X-*.md` 取一个素材条目（含 ID、尺寸、专属提示词）。
3. **拼提示词**：`{{STYLE}}` + 素材专属正文 + `{{NEG}}`（占位符定义见 00 号文件）。
4. **生成**：每个素材出 **4 个候选**。
5. **自检**：按 `QA-PROTOCOL.md` 的评分表给每个候选打分，输出规定的 JSON。
6. **判定**：达阈值 → 选最高分定稿；未达 → 用 QA 给出的 `fix_deltas` 改提示词，回到第 4 步（最多 4 轮，超出则上报人工）。
7. **整套门禁**：同一套（如 12 个 App 图标）各自通过后，做"contact sheet 合影评审"，确认像一家人。
8. **全局门禁**：把代表性素材合成进真实店铺场景 / 手机屏，做最终违和感复核。

## 锚点与不变量

- **锚点图**：`assets/art/realistic-shop-counter.png`（温暖手绘 2.5D，哑光、暖灰低饱和）。所有素材向它看齐，**不要反过来改它**。
- **统一光向**：所有"抠图类"素材（图标/顾客/弹窗/物件）共用**左上约 40° 柔和暖色主光**，使其成为一家人。
- **统一接地**：每个抠图对象自带一枚柔和模糊接地阴影，避免"漂浮贴片"。
- **文件落点**：定稿 PNG 覆盖 `assets/art/<id>.png`，文件名必须与 `_dev/src/content/asset-manifest.ts` 中的路径一致；同时同步到 `_dev/public/assets/art/`。音频落到 `assets/audio/` 并在 manifest 的 `audio` 字段接入。
