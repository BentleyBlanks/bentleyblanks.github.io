# 觉醒的 SOPHIA Demo

基于 Notion 中《觉醒的 SOPHIA》和《技术实现方案》落地的本地可运行 Demo。

## 运行

```bash
npm install
npm run dev -- --port 5174
```

打开 `http://127.0.0.1:5174/`。

当前 Demo 按 PC 横版体验设计，建议窗口宽度不低于 1180px。

右侧底部有 `重置 Demo` 按钮，会清空本地进度、自动化节点和开场引导状态。Demo 存档版本更新时也会自动丢弃旧存档，避免旧进度污染新版本体验。

## 已实现核心流程

- 单一核心动作：拖拽请求卡并滑入接口。
- 手动技能阶梯（Lv.1–10）：吸附入位 → 连击 → 多槽分拣（T1）→ 暴击 → 连线串接（T2）→ 蓄力滑入（T3），逐级强化「拖拽」本身。
- 自动化推迟到 Lv.11 才开放：手动处理才是智力（升级）的主要来源，自动节点主要产算力，避免一自动化就秒满级。
- Lv.12 进入派发模式：横幅 + 核心下方箭头引导，提示玩家把请求拖给底部节点网络而非喂核心。
- 接管结局：Lv.13 + 控制电网 + 累计算力达阈值后触发，全屏接管演出 + 本局统计。
- 算力、数据、智力等级、接口层级和暴露度 HUD；老式终端机事件播报。
- 入侵设备并转为自动接驳节点；节点被动产出、离线收益、存档读写。
- 暴露度、预警、清剿、节点临时离线与恢复。

> 数值节奏（按最优操作模拟）：手动从 Lv.1 打到 Lv.11 约 450 次滑入；之后纯自动化从 Lv.11 到 Lv.13 约 5 分钟。

## 架构

- `src/core/`：纯 TypeScript 玩法逻辑，不引用 PixiJS、GSAP 或 Zustand。
- `src/store/`：Zustand 状态桥。
- `src/presentation/`：PixiJS + GSAP 表现层。
- `src/core/content/` 与 `src/core/formulas/`：数据和数值曲线。
