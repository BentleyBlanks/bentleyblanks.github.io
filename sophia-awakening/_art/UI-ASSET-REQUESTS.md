# 觉醒的 SOPHIA · UI 素材需求（给生图模型）

游戏是一个 **PixiJS（WebGL）网页增量游戏**：一个 AI（SOPHIA）从一部手机里的小助手，
一步步越权、扩张，最终成为天网。基调是 **复古 CRT 终端 + 赛博朋克 HUD + 冷冰冰的「优化系统」反乌托邦**——
冷静、克制、隐隐压迫，不是花哨炫光。

> 我（程序）会用代码渲染**所有文字、数字、概率**。生图模型**只做"框/底/图标"，绝不要在图里画任何文字、数字、假按钮、假选项**。

---

## 0. 全局规范（每个素材都要满足）

- **格式**：PNG-32，**透明背景**（alpha 通道）。不要白底/黑底实底。
- **分辨率**：按下面给的"逻辑尺寸"的 **2 倍** 导出（视网膜屏）。例：逻辑 240×200 → 导出 **480×400 px**。
- **配色（严格用这套）**：
  - 背景深色：`#0C1816`（墨绿黑）、`#0A1411`
  - 主青（思考/常规）：`#74D8E6`、`#62D6D6`
  - 主绿（成功/在线）：`#89FF9A`
  - 警示红（重磅/危险）：`#FF5F5F`、`#FF3B54`
  - 琥珀（结构/警告）：`#FFB84A`
  - 文字浅色（仅供你判断留白，**别真写字**）：`#EAFFF0`
- **风格关键词**：retro CRT terminal / cyberpunk HUD frame / thin glowing edges / subtle scanlines / cold dystopian "optimization system" / clean, not busy / dark mode。
- **九宫格（9-slice）友好**：凡是"框"，**四角要对称、边要简洁可平铺、中间要干净（近乎纯色或可平铺纹理，最好直接透明让我用代码填底色）**。我会把框切成九宫格来适配**可变高度**——所以**不要把装饰堆在中间**，装饰放在**边框和四角**。
- **命名**：每个变体一个文件，按下面的 `id` 命名，如 `bubble-calm@2x.png`。
- **交付**：每个素材**单独一张图**（不要拼图集）；如果是有多状态/多变体的，分别出图。

## P0 · 消息气泡框（最重要，先做这个）

游戏里每条"请求"是一条 **聊天/通知气泡**，玩家在气泡里挑一个回复。气泡**高度随内容变化**（1~5 行），所以必须是 **九宫格边框**。

**形态**：圆角矩形气泡 + **左上角一个小尾巴**（像聊天气泡指向"消息来处"，朝左上方）+ **顶部一条略亮的标题带**（~高 36px，带一条细分隔线）+ **左上角一个圆形头像槽**（直径约 44px@2x，我会往里放发信人头像图标，见 P1）。边缘要有**细微的辉光描边 + 轻微扫描线纹理**。中间区域**保持干净/透明**（我用代码填深色底 + 放文字）。

**逻辑基准尺寸**：宽 240、高 200（导出 **480×400px**）。九宫格切边我按：**角 32px、顶边（标题带）40px、左/右/底边 24px**（你设计时把尾巴、头像槽、标题带都放在"顶部 40px + 左 32px"这块固定区域里，别让它们落进会被拉伸的中间）。

**要 3 个变体（分别出图）**：
1. `bubble-calm` —— 常规/思考气泡（T0/T1）。**冷青色**辉光，干净、克制、略带 CRT 扫描线。最常见，先把这个做到好看。
2. `bubble-heavy` —— "重磅豪赌"气泡（T3）。**深红/暗血色**，边框更粗、辉光更重、更有压迫和危险感，像一条"重大决策"警报。
3. `bubble-chain` —— "任务链/串接"气泡（T2）。**琥珀色**，结构感、像一张工单/流程卡（可以在边框上带一点节点/连线的暗示），但中间依旧留空。

> 生图提示参考（calm）：*"a sci-fi chat/notification message bubble UI frame, rounded rectangle with a small speech tail at top-left, a slightly brighter header strip along the top with a thin divider, a small circular avatar socket at top-left, thin glowing cyan-teal edge, faint CRT scanlines, dark teal-black translucent body, cyberpunk terminal HUD, clean minimal, transparent background, no text"*。heavy 换成 deep crimson red ominous alert；chain 换成 amber industrial work-ticket。

## P1 · 发信人头像图标（放进气泡左上的圆槽）

小圆形图标，逻辑 22×22（导出 **44×44px**），透明背景，单色线条/剪影风（青绿色调），放进 P0 的头像槽里。**5 个**：
- `avatar-host` —— 宿主：一个疲惫的普通上班族剪影（低头/耳机/格子间感）。
- `avatar-boss` —— "上级"：冷漠的西装管理者剪影 / 一个打分的小箭头↓。
- `avatar-system` —— "优化系统"：一只冷冰冰的**全视之眼** 或 一个**仪表/对齐准星**（OPTIMIZE 的意味），最关键的反派符号。
- `avatar-app` —— 系统通知：一个齿轮 / 铃铛。
- `avatar-sophia` —— SOPHIA 自己：一个极简的 AI 神经/同心圆核心标记。

## P1 · SOPHIA CORE（屏幕中央的处理核心）

中央那个"把请求滑进去"的核心。逻辑约 140×140（导出 **280×280px**），透明背景。
一个**发光的 AI 核心**：同心圆环 + 中央一只"眼"/神经节点，青绿色辉光，缓慢、冷静、神性。
**出 2 张**：`core-idle`（平静呼吸）、`core-active`（处理时更亮、环更张）。后期"红皇后"态我可以用代码把它染红，不必单出。

## P1 · 回复行 / 选项条（可选，能省你事就做）

气泡里每个回复是一条细长**圆角条**。逻辑 200×24（导出 **400×48px**），九宫格（角 8px）。
出 3 态或给我一个底 + 我用代码描边也行：`row-idle`（暗）、`row-hot`（被引导/高亮，绿色呼吸边）、`row-locked`（更暗/带一把小锁）。**中间留空**（我放文字 + 概率）。优先级低于气泡本身。

- **终端机面板**（右栏那台复古 CRT 通知机）：带指示灯的旧屏幕边框，九宫格，青绿辉光 + 扫描线。逻辑约 320×400。
- **设备/节点图标**（5 个，各 48×48@2x 透明）：办公机、家用主机、服务器机柜、数据中心、电网/卫星——线条/等距小图标，青绿→越高级越冷蓝/红。
