Unity Script Optimization

来自于Inside在Unite 2016大会上的性能优化报告

以下为演示Demo Code

public void func()
{
    var lastPos = transform.position;
    transforms.position = lastPos 
    + velocity * speed * factor 
    * Mathf.Sin(factor2) 
    * drag * friction * Time.deltaTime;
}

  1. Reduce Vector Operations

    为了减少Vector临时变量等生成，可以将所有标量先行算完，节省向量操作次数

     // ...
 transforms.position = lastPos 
 + velocity * (speed * factor 
 * Mathf.Sin(factor2) 
 * drag * friction * Time.deltaTime);

  2. Used cached Transforms
    Unity在使用transform时会做非常多的安全监测以保证其存在，因此可以在初始化时存下其引用以加快后期访问速度

     // _transform cached in OnStart()
 var lastPos = _transform.position;
 _transforms.position = lastPos + // ...;

  3. Use localPosition(if possible)
    一般更改Position时Unity会对其Hierarchy上的所有对象都进行位置更新，如若不需要这一步操作或其上的位置属性不会因此更改则可以使用localPosition

     // _transform cached in OnStart()
 var lastPos = _transform.localPosition;
 _transforms.localPosition = lastPos + // ...;

  4. Reduce engine calls
    应尽量减少对系统内变量的直接访问，可采用全局变量形式能减少engine call

     public Vector3 cachedLocalPosition = Vector3.zero;

 public void func()
 {
     cachedLocalPosition += velocity * speed * factor 
                          * Mathf.Sin(factor2) 
                          * drag * friction * Time.deltaTime;
     _transform.localPosition = cachedLocalPosition;       
 }

  5. Don’t do Vector math at all
    为了减少计算过程中Vector隐式转换带来的开销，这里应尽量使用分量赋值的形式来避免过多的隐含Vector生成的问题

     float factor = speed * factor 
              * Mathf.Sin(factor2) 
              * drag * friction * Time.deltaTime;
 cachedLocalPosition.x += velocity.x * factor;
 cachedLocalPosition.y += velocity.y * factor;
 cachedLocalPosition.z += velocity.z * factor;
 _transform.localPosition = cachedLocalPosition;

  6. Time.deltaTime could be cached
    预存下deltaTime，类似transform的思路

     public static float gloabalDeltaTime;
 public void func()
 {
     // change Time.deltaTime to gloabalDeltaTime
 }

    为更新所有脚本队列，因此事先预存globalTime。并做一系列优化

     public void Update()
 {
     for (int a=0; a<1000; ++a)
     {
         Test.gloabalDeltaTime = Time.deltaTime;
         foreach (var script in scriptList)
             script.Update();
     }
 }

  7. Don’t use foreach

     public void Update()
 {
     for (int a=0; a<1000; ++a)
     {
         Test.gloabalDeltaTime = Time.deltaTime;
         for (int i=0, count=scriptList.Count; i<count; ++i)
             scriptList[i].Update();
     }
 }

  8. Use Arrays instead of Lists

     public Test[] scriptArray;
 public void Update()
 {
     for (int a=0; a<1000; ++a)
     {
         Test.gloabalDeltaTime = Time.deltaTime;
         for (int i=0, count=scriptList.Count; i<count; ++i)
             scriptArray.Update();
     }
 }

  9. Use same lookup table sin instead of Mathf.Sin ensure different platforms’s result are the same

     // Size could set different depending on different platforms
 private static float[] sinLUT = new float[1024];
 public static float FastSin(float f)
 {
     int index = (int)(f * 1024) & 1023;
     return sinLUT(index);
 }
 public void Start()
 {
     float forwardStep = (2.0f * Mathf.PI) / (float)1024;
     for (int i=0; i<1024; ++i)
     {
         // Other way to compute sin() is okay
         float f = (float)i * forwardStep;
         sinLUT[i] = Mathf.Sin[f];
     }
 }

以上基本为脚本上的实现细节。同时也应注意
Don’t generate any garbage Keep an eye on GC Alloc
Don’t use Instantiate() runtime Instantiate object on first frame
Don’t do string concatenations Use StringBuilder if necessary
Built-in data structures perform horibbly Dicts have a huge performance spike first time they are referenced. Event Subscription is just expensive
Avoid using Update() and FixedUpdate() Every call from the Unity to Update() has a hidden performance overhead. Make your own Update-manager. In Inside this saved them 1-2ms per frame on XBox one and PS4
